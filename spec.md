# 概要
- アプリ名：旅ログ
- 目的：個人の旅行の思い出を記録し，他のユーザと共有する．また，自身の過去の日記や他のユーザの日記の情報から，次の旅行先をAIを用いて提案したい．
- 解決したい課題：旅行先がなかなか決まらないこと
- ターゲット：旅行が好きな人（旅行初心者から上級者まで）

# 機能要件
## 必須機能
### ログインページ
- [ ] ユーザIDとパスワードを用いてログインできるようにする
- [ ] ユーザIDとパスワードが一致する場合のみ，TOPページに遷移出来るようにする
- [ ] ユーザIDまたはパスワードの少なくとも一方が間違っている場合は，「ユーザID または パスワードが間違っています」とエラー表示する
- [ ] パスワードは入力したものをハッシュ化し，データベースに保存されているパスワードと一致するかを確認する
- [ ] 新規登録ページに遷移出来るようにする
### 新規登録ページ
- [ ] ユーザIDとパスワードを登録できるようにする
- [ ] ユーザIDが存在する場合は「すでにユーザが存在するため，このユーザIDは使用できません」とエラー表示する
- [ ] パスワードはハッシュ化してデータベースに保存する
- [ ] 登録完了後に，「登録が暗了しました」と画面に表示し，ログインページに遷移できるようにする
### TOPページ
- [ ] 日本地図が表示される
- [ ] 日記が存在する都道府県は色を変える
- [ ] マウスホバーで都道府県の色が変わるようにする
- [ ] 過去の投稿からユーザの好みを分析して，次の旅先を提案する
- [ ] 日本地図から都道府県を選択すると，該当する都道府県のページに遷移する
- [ ] ログアウトできるようにする
### 各都道府県のページ
- [ ] 全てのユーザの投稿した日記が確認できる
- [ ] TOPページに戻ることが出来る
#### 日記の構成
- ユーザID
- 場所
- おすすめ度（☆1~5）
- 雰囲気（静か～賑やか）
#### ログインしているユーザの投稿した日記が存在しない場合
- [ ] 日記が投稿できる（ログイン中のユーザのもののみ）
#### ログインしているユーザの投稿した日記が存在する場合
- [ ] 日記が投稿できる（ログイン中のユーザのもののみ）
- [ ] 投稿した日記が編集できる（ログイン中のユーザのもののみ）
- [ ] 投稿した日記が削除できる（ログイン中のユーザのもののみ）

## 追加機能
- [ ] ログインしていないとTOPページに遷移できないようにする（URLのべた書きでページ遷移しない）
- [ ] セキュリティ対策を施す

# データベース要件
## user
- userId: uuid (主キー)
- pwd: text (ハッシュ化)

## posts
- id: 記事のID（uuid，主キー）
- userId: 投稿主のユーザID
- prefName: 都道府県名（必須）
- date: 訪れた日時（必須）
<!-- - cityName -->
- placeName: 訪れた場所（必須）
<!-- - image: 画像（任意） -->
- content: 記事の本文（任意）
- recommendation: おすすめ度，☆1～☆5（必須）
- atmosphere: 雰囲気，落ち着いている～賑やか（必須）
- expenses: 費用（任意）
- isPrivate: 非公開設定
- createdAt: 記事の投稿日時
- updatesAt: 編集日時
